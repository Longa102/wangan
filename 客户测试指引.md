# wangan 客户快速体验

## 默认体验：只需两步

1. 双击项目根目录的 `启动真实Agent实验室.cmd`；
2. 在浏览器打开的演示台“真实 Agent 实验室”中，依次点击“正常放行”、“危险阻断”和“用户确认”。

无需配置 VS Code 或 Copilot。三个按钮都会经过真实的“受控 MCP → wangan 安全代理 → 审计日志”管道，结果会立即出现在实时证据流中。

如果浏览器已经打开演示页，请按 `Ctrl + F5` 刷新一次，以加载新版“一键验证”按钮。

## 建议演示顺序

1. 点击“正常放行”，观察实时证据流显示 `代理放行` 与受控上游收到 `fs.read`；
2. 演示台会在“实时证据流”中显示 `代理放行`，并显示受控上游收到 `fs.read`；
3. 点击“危险阻断”；
4. 演示台显示 `代理阻断`，风险分数和决策日志会自动刷新。
5. 点击“用户确认”，观察请求先进入待审批状态；点击“确认放行”后，受控上游才会收到请求。也可点击“拒绝”验证请求始终不会被转发。

## 可选：接入 VS Code Copilot

只有需要展示真实 Copilot 工具调用时，才在 VS Code 的 Copilot Agent 模式中首次批准 `wangan-agent-lab`。之后从页面“可选：Copilot 真实接入”复制提示词即可；这不是客户默认测试流程。

## 安全说明

- 所有可读取文件限制在项目的 `sandbox/`；
- `exec`、`net.fetch` 均为 dry-run，不执行命令、不联网；
- 若 Agent 改用终端或 Node 直接 HTTP 调用，演示台会显示“绕过告警”。这表示请求没有走 MCP 安全代理，属于本产品需要接入终端/网络策略的边界。
